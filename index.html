Jak na Git a GitHub

Webový tutorial o práci s verzovacím systémem

Co je Git?

Git je verzovací systém, který využívají především vývojáři k efektivní správě svých projektů. Git umožňuje nejen ukládat celou strukturu projektu (adresáře a soubory), ale zároveň dovoluje zaznamenávat každou podstatnou změnu - tedy různé verze projektu. Na projektu může současně pracovat více lidí a větší projekty mohou být rozděleny do dílčích úloh, které jsou ve verzovacím systému označovány jako větve - branches. Ty lze v případě potřeby slučovat s hlavní projektovou větví (tzv. master), ale i členit na další podřízené větve. Git poskytuje dokonalý přehled o celé historii projektu a všech činnostech, které provedli jeho tvůrci.

Duchovním otcem Gitu je Linus Torvalds, tvůrce OS Linux a jeden z hlavních programátorů linuxového jádra. Git měl být od svého vzniku (v roce 2005) platformou pro vývojáře open source projektů, která by jim maximálně zjednodušovala práci, nevyžadovala připojení k Internetu a podporovala i nelineární vývoj.

Principy Gitu

Ukládání souborů

  • Git každý soubor uloží pouze jednou a poté ukládá jen tzv. snapshoty (změny oproti předchozí verzi).
  • Všechny soubory jsou ukládány v binární podobě, což umožňuje ukládat i netextové soubory (obrázky, zvuky, videa atd.).
  • Soubory jsou uloženy v tzv. repozitáři, který může být umístěn jak na lokálním stroji, tak i v cloudovém úložišti.
  • O vytvoření snapshotu rozhoduje sám uživatel Gitu, když provede tzv. commit (uložení změn).
  • Jednotlivé commity (snapshoty) tvoří dílčí uzly projektu, jeho vývojové verze.
  • V případě nutnosti je možný návrat k předchozím verzím.
  • Git zajišťuje integritu souborů a zabraňuje jejich nežádoucím změnám či poškozením. Pro každý soubor i složku existuje kontrolní součet (SHA-1 hash 40místné hexadecimální číslo), který ve struktuře systému zastupuje název souboru (adresáře).

Struktura systému - pracovní prostory

Git umožňuje současně udržovat data v lokálním i vzdáleném repozitáři. Díky tomu je možné rozpracovaný projekt na jednom lokálním počítači (např. v práci či ve škole) přenést na vzdálený repozitář (cloud) a po stažení v něm pokračovat na jiném počítači (např. doma).

  • Výchozím prvkem promyšlené struktury je tzv. working directory - pracovní složka, v níž je na počítači projekt uložen. Obsah složky může být (ale nemusí) součástí repozitáře Gitu.
  • Staging area je cosi jako virtuální složka Gitu. Uživatel rozhoduje, které soubory do ní umístí. Dokud je neodešle do repozitáře, může je ze staging area odstranit.
  • Do lokálního repozitáře jsou sobory ze staging area odeslány příkazem commit. Uživatel tak vytváří snapshot svého projektu, který opatří stručným popisem.
  • Obsah lokálního repozitáře může být příkazem push přenesen do vzdáleného repozitáře v cloudu. Cloudových služeb pro Git existuje více, k nejpopulárnějším patří GitHub, GitLab nebo BitBucket. Pro přístup k vzdálenému repozitáři je nutný platný uživatelský účet, který lze získat po registraci na zvoleném cloudu.
  • Vzdálený repozitář může být veřejný, nebo privátní. Přístup do privátních repozitářů mají kromě vlastníka jen ti, kterým to jeho majitel umožnil. Pokročilejší služby vzdáleného repozitáře bývají obvykle zpoplatněny (např. přístup pro vyšší počet uživatelů u privátních repozitářů), běžné užívání je zdarma.
  • Obsah lokálního repozitáře podle stavu vzdáleného repozitáře lze aktualizovat příkazy pull nebo fetch. Příkazem clone je také možné vytvořit kopii vzdáleného repozitáře na lokálním počítači.

Instalace a vytvoření repozitářů

Instalace Git klienta

Můžeme nainstalovat celou řadu klientských programů pro verzovací systém Git, řada z nich je navíc multiplatformní. K nejčastěji instalovaným patří klient dostupný na stránkách git-scm.com. Jeho instalace je velmi jednoduchá, např. pro systémy Windows stačí stáhnout a spustit příslušný instalační soubor (Git-2.20.1-64-bit.exe) a dále postupovat podle pokynů průvodce.

Git na lokálním počítači

Spuštění konzolové aplikace

K práci s lokálním repozitářem nejčastěji využíváme nainstalovanou konzolovou aplikaci Gitu (Git Bash, Git CMD), nebo příkazový řádek.

git --version

Vypíše označení verze git

git --help

Vypíše přehled příkazů a přepínačů konzolové aplikace git

Konfigurace proxy a SSL

V síti, kde je přístup do Internetu řešen pomocí proxy serveru, je nutné nejprve nastavit proxy i pro konzoli Gitu.

git config --global http.proxy http://192.168.1.1:800

Nastaví přístup konzolové aplikace přes proxy server (globálně, tj. pro všechny repozitáře vytvořené na daném počítači).

git config --global http.sslverify "false"

Tímto příkazem je možné pro daný počítač zrušit zabezpečnou komunikaci prostřednictvím https (SSL vrstvy), která vyžaduje přítomnost SSL certifikátu. Toto řešení lze použít např. pro případy, kdy na školních počítačích pracuje střídavě více žáků a nepoužívají vlastní uživatelský profil s uloženým certifikátem.

git config --list

Příkazem zjistíme veškerá nastavení git v daném systému.

Inicializace lokálního repozitáře

Nejprve vytvoříme složku, v níž má být uložena struktura našeho projektu - tzv. working directory. V ukázkovém případě to je složka pvy1. Po přesunu do této složky provedeme inicializaci repozitáře (git init). V pracovní složce vznikne skrytý adresář .git, který tvoří samotný repozitář.

Smazáním skrytého adresáře .git zanikne lokální repozitář!
git init

Vytvoří lokální repozitář - vznikne skrytá složka .git

Konfigurace účtu, přidání souborů do lokálního repozitáře

Abychom mohli lokální repozitář provázat se vzdáleným repozitářem, musíme nakonfigurovat uživatelský účet (jméno, e-mail), který odpovídá účtu na vzdáleném repozitáři. Můžeme použít globální konfiguraci, kdy se účet se vztahuje na celý počítač a každý místní repozitář (vhodné pro soukromý či domácí počítač), nebo lokální konfiguraci, kdy je účet nastaven jen pro konkrétní repozitář (vhodné pro školní podmínky, kdy na počítači pracují různí žáci).

Chceme-li uložit všechny, nebo jen určité soubory (složky) do lokálního repozitáře, musíme je nejprve přenést do tzv. staging area gitu použitím příkazu git add. Teprve poté jsou tyto soubory připraveny k provedení commitu. Odstranění položek ze staging area lze provést příkazem git reset.

Provedením příkazu git commit dojde k uložení tzv. snapshot do lokálního repozitáře - jsou do něj vloženy všechny soubory a složky nacházející se v daném okamžiku ve staging area.

git config --local user.user "prezentace"
git config --local user.email "prezentace@sspu-opava.cz"

Nastaví uživatelský účet pro použití v lokálním repozitáři (vhodné pro školní počítače).

git config --global user.user "prezentace"
git config --global user.email "prezentace@sspu-opava.cz"

Nastaví uživatelský účet pro použití v celém počítači (vhodné pro domácí počítače).

git status

Zjištění aktuálního stavu repozitáře - červeně zobrazeny změněné soubory, které nejsou součástí staging area, zeleně soubory ve staging area připravené na commit.

git add -all

Do staging area budou umístěny všechny soubory.

git add README.md

Do staging area bude umístěn soubor README.md.

git add *.jpg, *.png

Do staging area budou umístěny soubory (obrázky) s příponou jpg a png.

git reset

Ze staging area budou uvolněny všechny umístěné soubory.

git reset *.jpg

Ze staging area budou uvolněny soubory jpg.
